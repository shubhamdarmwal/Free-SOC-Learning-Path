# Investigating with ELK 101

**Room:** [tryhackme.com/room/investigatingwithelk101](https://tryhackme.com/room/investigatingwithelk101)

ELK is another SIEM stack used heavily in SOCs, especially in open-source/self-hosted setups. As an analyst, you mainly live in Kibana — don't need to deep dive into how each component works internally.

---

## Elastic Stack Components

1. **Elasticsearch** — stores and indexes the log data, makes it searchable
2. **Logstash** — processes and transforms logs before they're stored (parsing, filtering, enriching)
3. **Beats** — lightweight agents installed on endpoints that ship logs to Logstash/Elasticsearch
4. **Kibana** — the UI/dashboard layer analysts actually use to search and visualise

```
Endpoint (Beats) → Logstash (processing) → Elasticsearch (storage/index) → Kibana (analyst UI)
```

---

## Discover Tab

This is the main workspace for exploring raw logs.

- **Logs** — each row is one event with its fields and values
- **Fields Pane** — left panel showing all parsed fields, click to filter by them
- **Index Pattern** — different log types live in different indexes (e.g. VPN logs in a `vpn_connections` index) — pick the right one
- **Search Bar** — where you write KQL queries
- **Time Filter** — narrow results to a specific time range
- **Time Interval Chart** — shows event count over time
- **Add Filter** — apply field filters without writing a full query

**Lab:** queried the `vpn_connections` index, filtered between 31 Dec 2021 and 2 Feb 2022, and answered questions based on the results.

## KQL (Kibana Query Language)

Two ways to search:

**Free text search** — just type a keyword and it searches across all fields
```
"United States"
```

**Field-based search** — target a specific field
```
event.outcome: "failure"
```

**Logical operators:**
```
event.outcome: "failure" AND source.ip: "10.0.0.5"
event.outcome: "failure" OR event.outcome: "blocked"
NOT user.name: "admin"
```

## Visualizations

Built a visualization for **Failed Connection Attempts** to spot patterns in VPN login failures.

Steps to save a visualization:
1. Create the visualization
2. Click **Save** (top right)
3. Add title and description
4. Save and add to library

**Lab:** answered questions based on the Failed Connection Attempts visualization — things like which user/IP had the most failures, time patterns etc.

---

## Dashboards

Dashboards combine multiple visualizations and saved searches into one view.

Steps:
1. Go to Dashboard tab → Create dashboard
2. Click Add from Library
3. Add visualizations/saved searches
4. Arrange them
5. Save the dashboard
